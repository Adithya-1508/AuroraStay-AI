# Spec: Reservation Domain Events

- **Status**: Draft
- **Owner**: Domain Engineering Team (Antigravity AI Coding Agent)
- **Date**: 2026-07-06

## 1. Purpose
Define the events generated by the reservation system for consumption by future asynchronous business domains (e.g. Housekeeping, Billing, Analytics).

## 2. Responsibilities
- Model standard event structures.
- Enforce timestamps and sequence validation.
- Provide a publication handler interface.

## 3. Dependencies
- **Shared Utils**: For standard JSON serialization and base event class.

## 4. Interfaces
### Event Schemas
```python
class ReservationBaseEvent(BaseModel):
    event_id: UUID = Field(default_factory=uuid4)
    timestamp: datetime = Field(default_factory=lambda: datetime.now(UTC))
    version: int = 1

class ReservationCreated(ReservationBaseEvent):
    reservation_id: UUID
    guest_id: UUID
    room_category_id: UUID
    check_in_date: date
    check_out_date: date
    total_cost: Decimal

class ReservationUpdated(ReservationBaseEvent):
    reservation_id: UUID
    old_check_in: date
    new_check_in: date
    old_check_out: date
    new_check_out: date
    old_total_cost: Decimal
    new_total_cost: Decimal

class ReservationCancelled(ReservationBaseEvent):
    reservation_id: UUID
    reason: str

class ReservationConfirmed(ReservationBaseEvent):
    reservation_id: UUID
    assigned_room_id: UUID

class ReservationCheckedIn(ReservationBaseEvent):
    reservation_id: UUID
    assigned_room_id: UUID

class ReservationCheckedOut(ReservationBaseEvent):
    reservation_id: UUID

class ReservationCompleted(ReservationBaseEvent):
    reservation_id: UUID
```

## 5. Configuration
- `EVENT_BUS_DUMMY_MODE`: Default True (event execution writes to log stream for current loop, no external message broker required).

## 6. Error Handling
- Publication failure should not rollback the primary database transaction but must be logged for retry operations.

## 7. Security
- Events payload must not contain plain text guest credentials or secrets.

## 8. Testing
- **Unit Tests**:
  - Verify serialization and deserialization of each event model type.
  - Assert correct schema properties exist (e.g., date formats, UUIDs).

## 9. Acceptance Criteria
- [ ] Events are dispatched upon status changes in the reservation database.
- [ ] Events schema validation prevents publication of malformed message payloads.
